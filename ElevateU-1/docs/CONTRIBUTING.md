# Contributing to ElevateU

## Branching
- The main branch is protected.
- Feature branches should be named as `feature/<short-description>`.

## Commits
- Use conventional commits: `feat`, `fix`, `chore`, `docs`, `refactor`, `test`.

## Pull Requests
- Open pull requests early and request reviews.
- Fill out the pull request template and include testing steps.

## Code Style
- Python: Use `black`, `isort`, and `flake8` for code formatting and linting.
- JavaScript/TypeScript: Use `prettier` and `eslint` (to be added in the frontend).

## Environment
- Copy `.env.example` to `.env` and fill in the required secrets.
